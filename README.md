# VIN-декодер бот (v1.0.0)

Телеграм-бот для базовой проверки VIN и извлечения ключевой информации:
- страна производства (по 1-му символу VIN),
- производитель/завод (по WMI),
- модельный год (по 10-му символу VIN).

Проект построен на `aiogram 3` и использует собственный модуль валидации VIN.

## Важно про точность данных

Бот даёт **приблизительную справочную расшифровку** VIN на основе встроенных таблиц соответствий.
Точный состав и интерпретация некоторых полей могут отличаться у разных автопроизводителей и заводов,
потому что внутренние правила маркировки зависят от бренда, рынка и конкретной модели.

Поэтому результат бота стоит использовать для предварительной проверки, а для юридически/технически
критичных задач сверять данные по официальным источникам производителя и документам автомобиля.

## Что умеет версия 1.0.0

- Обрабатывает команду `/start`.
- Принимает VIN из обычного текстового сообщения.
- Нормализует ввод (`strip()`, `upper()`).
- Валидирует VIN по базовым правилам ISO-3779:
  - длина ровно 17 символов,
  - допустимы только символы из набора VIN (без `I`, `O`, `Q`),
  - VIN не может состоять только из цифр,
  - VIN не может состоять только из букв.
- Разбирает VIN на части:
  - `WMI` (1-3),
  - `VDS` (4-9),
  - `VIS` (10-17).
- Возвращает результат в формате:
  - валидный VIN -> краткая расшифровка,
  - невалидный VIN -> список найденных ошибок.

## Стек

- Python 3.11+
- `aiogram==3.27.0`
- `python-dotenv==1.2.2`

Полный список зависимостей: `requirements.txt`.

## Структура проекта

```text
checker/
├── bot.py                    # точка входа Telegram-бота
├── main.py                   # локальный CLI-тест валидатора
├── requirements.txt
└── core/
    ├── data/
    │   └── wmi_manufacturers.json # справочник WMI -> производитель
    ├── handlers/
    │   ├── start.py          # /start
    │   └── vin.py            # обработка VIN из сообщений
    ├── logging/
    │   └── logger.py         # настройка логирования
    └── utils/
        ├── constants.py      # коды стран, WMI и модельных годов
        ├── global_configs.py # ParseMode и общие настройки
        └── validator.py      # логика валидации VIN
```

## Быстрый старт

1. Клонируйте репозиторий и перейдите в папку проекта.
2. Создайте виртуальное окружение и активируйте его.
3. Установите зависимости:

```bash
pip install -r requirements.txt
```

4. Создайте файл `.env` в корне проекта:

```bash
cp .env.example .env
```

5. Добавьте токен бота в `.env`:

```env
BOT_TOKEN=your_telegram_bot_token_here
```

6. Запустите бота:

```bash
python bot.py
```

Если `BOT_TOKEN` не задан, приложение завершится с ошибкой:
`BOT_TOKEN не найден в .env!`.

## Использование

1. Откройте чат с ботом в Telegram.
2. Отправьте `/start`.
3. Отправьте VIN строкой, например:

```text
1HGCM8263TA004352
```

### Пример успешного ответа

```text
✅ 1HGCM8263TA004352

– Страна: США
– Завод: Honda
– Модельный год: 2026
```

> Примечание: фактический результат зависит от таблиц `YEAR_CODES`, `COUNTRY_CODES` и `WMI_MANUFACTURERS` в `core/utils/constants.py`.

### Пример ответа с ошибками

```text
❌ VIN не прошёл проверку:

• Количество знаков VIN некорректно (16).
• Найдены некорректные символы: ['I'].
```

## Локальная проверка без Telegram

Если нужно проверить логику без Telegram, используйте локальный консольный режим:

```bash
python main.py
```

Команда запросит VIN в терминале. Для выхода введите `exit` или пустую строку.

## Ограничения текущей версии

- Валидация проверяет базовые требования и справочники, но не считает контрольную сумму VIN.
- Поддержка производителей и кодов ограничена словарями в `constants.py`.
- Справочник модельных годов сейчас заполнен до `2026`.

## Идеи для v1.1+

- Добавить проверку контрольной суммы VIN.
- Расширить словари `WMI_MANUFACTURERS` и `YEAR_CODES`.
- Добавить тесты (`pytest`) для валидатора и хендлеров.
- Добавить Dockerfile для простого деплоя.
